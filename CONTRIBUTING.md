# Contributing prompts

Thank you for adding to the library. Two rules keep it safe and useful:

1. **No client, personal or sensitive data — ever.** Use placeholders like
   `{CLIENT}`, `{NAME}`, `{SYSTEM}`, `{DATE}`. No real names, IDs, ticket or
   incident numbers, internal system/table/repo names, or account specifics.
2. **Make it reusable.** A good prompt has a clear *title*, a one-line *use when*,
   a *system prompt* (role + rules), and a *user template* with placeholders.

Submit via the **＋ Add prompt** button on the site. Your prompt is shared with
everyone but stays hidden until a maintainer approves it.

Maintainers: review for the data rule above before approving.
